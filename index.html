Procedural Terrain Generation System

This project is a Procedural Terrain Generation System built in Unity, designed to create natural-looking, dynamic landscapes entirely at runtime. The system generates terrain by combining noise-based heightmaps, chunk-based loading, and real-time mesh generation, making it scalable for large open-world environments. It features chunk-based terrain that efficiently generates and streams terrain in sections for infinite worlds. The terrain shapes are driven by Perlin noise and other algorithms for realistic results. Parameters such as scale, height, and noise values can be adjusted in real time. Level of Detail (LOD) dynamically adjusts mesh detail based on the playerâ€™s distance to optimize performance, and all chunks align seamlessly without visible seams. The system supports customizable materials for varied biomes and is performance-oriented to ensure smooth generation even in large environments. It was developed using Unity as the game engine and C# as the primary scripting language. Terrain shapes are created using Perlin Noise and Simplex Noise, with meshes generated through direct manipulation of vertices, triangles, and UVs via the Unity Mesh API. This system can be used in open-world adventure or survival games, strategy games with dynamically generated maps, and simulation environments requiring varied terrain layouts. Planned future improvements include adding rivers, lakes, and other natural features, implementing biome-based generation with vegetation placement, integrating erosion simulation for more natural terrain shapes, and adding multiplayer streaming support for networked worlds.

Procedural Dungeons Generation System

The procedural dungeon generation system is designed to create unique, non-repetitive dungeon layouts at runtime, ensuring every playthrough offers a fresh experience. It uses a Binary Space Partitioning (BSP) algorithm to divide the map into rooms of varying sizes, then connects them with hallways using a combination of Delaunay triangulation and a Minimum Spanning Tree (MST) to ensure optimal connectivity without unnecessary paths. Room placement is flexible, allowing for irregular layouts rather than rigid grids, and hallway generation uses a pathfinding approach to navigate around obstacles. The system supports dynamic room scaling, variable door placement, and mesh-based wall generation with separate colliders for walls and doorways. This architecture enables both performance optimization and high visual fidelity, making it suitable for immersive exploration, combat encounters, and adaptive level design.
